## Overview

Meta.Instruct documents provide architectural understanding of each subsystem in 20-50 lines. They answer: **What does this subsystem do? How does it work? What are its critical patterns?**

Use meta.instruct for **architectural overview**. Use MODULE.md for **component details**. Use comprehensive guides for **deep dives**.

## When to Consult Each

**Meta.Instruct** (this layer):
- Understanding subsystem architecture
- Learning how components interact
- Finding integration points
- Quick reference for common operations

**MODULE.md** (component level):
- Component-level implementation details
- API documentation
- Configuration options
- Class/function signatures

**Comprehensive Guides** (deep dives):
- Step-by-step tutorials
- Advanced customization
- Performance optimization
- Troubleshooting
